Адаптивный frontend-прототип веб-приложения для учёта IT-инфраструктуры: рабочие места, кассовое оборудование, принтеры.

## О проекте

В IT-отделе учёт оборудования вёлся в разрозненных Google-таблицах. Прототип **Control Panel** объединяет ключевые сценарии в одном интерфейсе:

- **Рабочие места** — сотрудник, доменное имя ПК, характеристики, статус «Активно / Неактивно»
- **Кассы** — контроль сроков **ФН** и **ОФД**, организация, терминал
- **Принтеры** — тип (МФУ / термо / лазер), подключение **сеть / USB**, IP с копированием в буфер
- **Обзор** — сводка и блоки «требуют внимания» (просрочки, кассы без терминала, принтеры без IP)

Справочник сотрудников и компьютеров имитирует выгрузку **Active Directory** (на стенде — демо-данные).

**Компетенции практики:** ПК 8.1–8.3 (UX/UI-концепция, требования из предметной области, адаптивный frontend-прототип).

## Возможности

| Раздел 	| Функции|
|	---	|	---	|
| Обзор    	| Счётчики, срочная / ближайшая замена ФН и ОФД |
| Рабочие места | Группировка город → локация, карточки, перенос, добавление с выбором из AD |
| Кассы 	| Таблица / карточки на mobile, цветовая индикация сроков |
| Принтеры	| Сеть (IP) и USB, копирование IP |
| Общее 	| Поиск, фильтры по городу и организации, светлая и тёмная тема |

## Технологии

- HTML5, CSS3 (адаптивная вёрстка)
- JavaScript (ES-модули)
- `localStorage` + JSON-файл (`data/inventory-store.json`)
- Локальный HTTP-сервер на Node.js (`scripts/serve.mjs`)

Backend, авторизация и роли в прототип не входят — это следующий этап после согласования макета.

---

## Требования

- [Node.js](https://nodejs.org/) 18+ (достаточно для `node scripts/serve.mjs`)
- Современный браузер (Chrome, Edge, Firefox)

---

## Запуск

```bash
# из корня репозитория
node scripts/serve.mjs
```

Открыть в браузере: **http://127.0.0.1:8770**

Сервер сохраняет правки в `data/inventory-store.json`. Без сервера данные остаются только в `localStorage` браузера.
